Для того чтобы войти в аккаунт гитхаб через терминал есть два подхода
    1. Перейти в дисперчер учетных данных или в связку ключей на mac создать учетку
    для адреса git:https://github.com , далее прописать username аккаунта 
    а в качестве пароля сгенерировать токен на гитхаб
    
    2. Через SSH 
    Для подхода необходимо сгенерировать ssh ключей
    Команды: 
        cd ~
        ls
        (Если папка .ssh существует идем в нее пропуская следующую команду)
        mkdir .ssh
        cd .ssh
        ssh-keygen -o
        ls 
        (Если после команды видим <наш_ключ>.pub)
        cat <наш_ключ>.pub
        (Далее копирум ключ из терминала)
        (Переходим в гитхаб в настройки в ssh и там регистрируем ключ)


Команды для выгрузки репозитория на github
Добавление удаленного репозитория
    git remote add origin <Url до репозитория>(если зашли через учетные данные то вставляем 
        http адрес, если через ssh то ssh адрес)
        В данной команде "add" означает что добавляем удаленный репозиторий
        "add" можно заменить на "set-url" если хотите изменить уже добавленный адрес репозитория
        "origin" - имя удаленного репозитория с которым мы будем работать

Добавление коммитов в удаленный репозиторий
    git push origin master
        В данной команде "origin" название удаленного репозитория(куда добавляем)
        "master" та ветка которую добавляем

Клонирование удаленного репозитория 
    git clone <url адрес репозитория>

Обновление локального репозитория из глобального 
    git pull origin master (обнвляем только определенную ветку)
    git fetch origin (обновляем все ветки)

Переименование удаленных веток
    git push origin <старое название ветки>:<новое название ветки>(скопируем все из старой ветки в новую)
    git push origin :<название удаляемой ветки> (удаляем ветку из удаленного репозитория)

Откат к ранним комитам
    git reset --hard <хэш коммита> 
    git push -f <удаленный репо> <ветка которую обновляем>
